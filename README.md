# Табель — учёт личного состава подразделения

Офлайн-приложение для командира: учёт личного состава, строевая записка,
отпуска/командировки, выслуга лет и справочный расчёт денежного довольствия.
Стиль — «тактический графит» (тёмный фон + зелёный акцент, скевоморфные панели).

**Все данные хранятся только на устройстве** (SQLite через Drift) — без сервера,
интернета и регистрации. Подходит под любое подразделение (структура штата
редактируется).

> ⚠️ **Демо-версия.** Репозиторий содержит только **условные** данные
> (нейтральные фамилии для примера). Реальные персональные данные не входят и
> вводятся пользователем на своём устройстве.

Flutter (stable) · Android · Drift (SQLite) · Riverpod · go_router.

## Возможности

- **Расход ЛС** — строевая записка: по штату / налицо / по списку, разбивка по
  статусам и категориям, дни рождения месяца; экспорт PDF / Excel / текст.
- **Состав** — гибкий штат (любые должности и взводы), фильтры, поиск,
  массовая смена статуса с отменой (Undo), на карточке — выслуга.
- **Профиль** — контракт (подсветка сроков), звание, награды (гос. +
  ведомственные Росгвардии), квалификация, семья.
- **Отпуска / Командировки** — автосинхрон из статуса, лимиты, контроль возврата.
- **Выслуга лет** — календарная и льготная (день за 1,5 / 2 / 3 для СВО/боевых
  и отдельных местностей), надбавка за выслугу, право на пенсию (20 лет).
- **Калькулятор ДД** — оклад по тарифному разряду и званию + надбавки (выслуга,
  классность, особые условия, гостайна, физподготовка, риск, премия, районный
  коэффициент); суммы **до и после НДФЛ**; отдельная вкладка «Быстрый расчёт».
  Справочник окладов редактируемый.
- **Безопасность** — PIN (PBKDF2 + соль, лимит попыток); резервная копия
  устройства отключена; передача данных между устройствами файлом `.tbl`.

> Расчёт денежного довольствия — **справочный, оценочный**. Официальный расчёт
> выполняет финансовый орган; оклады и проценты надбавок следует сверять с
> действующими приказами.

## Запуск

```bash
flutter pub get
dart run build_runner build      # кодогенерация Drift
flutter run                      # или: flutter build apk --release --split-per-abi
```

Требуется Flutter (stable) и Android SDK. Сборка настроена под Android.

## Архитектура

`lib/`: `core/` (тема, виджеты, роутер), `data/` (Drift БД, таблицы, сид),
`domain/` (сущности, сервисы: выслуга, ДД, отпуска, штат), `presentation/`
(экраны + Riverpod-провайдеры). Тесты — в `test/` (домен-логика + смоук экранов).

## Данные

Первый запуск заполняется **демонстрационным** штатом (условные фамилии) — его
можно полностью очистить в настройках. Свои данные пользователь вводит вручную
или переносит экспортом/импортом `.tbl`.
