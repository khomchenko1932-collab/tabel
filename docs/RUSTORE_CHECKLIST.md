## Сборка релиза

**Сначала поднять build number** в pubspec.yaml: `version: 2.0.0+2003`.
Причина: тестовая сборка на телефоне ставилась через `--split-per-abi`,
её versionCode = 2002 (Flutter добавляет 2000 для arm64). Universal APK
с кодом 2 поверх неё не встанет (Android сочтёт это downgrade). Код 2003
решает проблему и остаётся монотонным для будущих релизов.

```bash
flutter build apk --release        # universal APK — его и загружать в RuStore
```

`--split-per-abi` для стора НЕ использовать: он даёт несколько APK со
смещёнными versionCode (1002/2002/3002), а RuStore принимает один файл.

Проверить, что подпись НЕ debug: `apksigner verify --print-certs <apk>`
(в выводе не должно быть CN=Android Debug).

## Карточка приложения в RuStore (факты сверены с оф. документацией 2026-07-18)

- Регистрация: console.rustore.ru через VK ID → «Физическое лицо» —
  МГНОВЕННО, без модерации заявки; паспорт-верификация не нужна
  (только для монетизации). ИНН не требуется.
- Загружать: ОДИН универсальный APK (`flutter build apk --release`).
  AAB в RuStore требует передать ЗИП с ключом подписи — нам не нужно.
- Название: до 30 символов. «Табель — учёт личного состава» = 29 ✅
  (если занято — «Табель ЛС», «Табель: учёт л/с»)
- Тексты: готовы в RUSTORE_CARD.md (короткое ≤80, полное ≤4000, «Что нового»)
- Категория: Органайзеры (запасная — Бизнес)
- Возрастной рейтинг: 0+ по 436-ФЗ (текст/таблицы, без опасного контента)
- Иконка карточки: 512×512 PNG ≤3МБ — ГОТОВА: docs/store_icon_512.png
- Скриншоты: мин. 1, до 10; соотношение 9:16; до 2160×3840; до 3МБ;
  реальный интерфейс, ДЕМО-данные, подразделение нейтральное («1 рота»)
- Контакты: достаточно email
- Политика конфиденциальности: обязательного поля URL нет, но страницу
  иметь стоит (модератор может запросить) — текст в PRIVACY_POLICY.md
- Анкета данных: «не собирает и не передаёт», разрешений ноль
- Модерация: обычно 1–5 рабочих дней (чаще 2–3)
- Позиционирование: нейтральное — «органайзер, учёт личного состава
  подразделения», данные вносит пользователь, всё офлайн. Без привязки
  к конкретным частям.
- Иконка: 512×512 PNG (есть: assets/icon/icon.png)
- Анкета о данных: «данные не собираются и не передаются» (офлайн)
- Email поддержки: khomchenko1932@gmail.com
